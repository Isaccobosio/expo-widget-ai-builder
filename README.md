# Fatture in Cloud — Widget POC (Expo SDK 56)

Proof-of-concept: a native home-screen widget for **Fatture in Cloud**, built with **Expo SDK 56** and the **official [`expo-widgets`](https://docs.expo.dev/versions/latest/sdk/widgets/) + [`@expo/ui`](https://docs.expo.dev/versions/latest/sdk/ui/)** modules — no hand-written Swift/Kotlin, no third-party widget libraries.

The widget renders the current-month revenue (**fatturato**) and the next couple of upcoming invoices/tax deadlines. Data comes from a local **MCP mock server** so the POC never touches the real Fatture in Cloud API.

## Requirements

- Node.js >= 20.19.4 (RN 0.85 engine hint; 20.19.3 also works with a warning).
- macOS with Xcode + iOS Simulator for `run:ios`.
- Android SDK + emulator for `run:android`.

## Setup

```bash
# 1. Install app deps
npm install

# 2. Install mock server deps
npm --prefix mcp-server install

# 3. Sanity checks
npx expo-doctor       # 18/18 pass
npx tsc --noEmit      # clean
npm --prefix mcp-server run test:tools   # dumps mock payloads
```

---

## Run

Open **two terminals**:

```bash
# Terminal 1 — start the HTTP mock the app fetches from
npm run mock:server                # http://localhost:4599

# Terminal 2 — regenerate native folders and run on the simulator
npm run prebuild                   # regenerates ios/ + android/
npm run run:ios                    # iOS Simulator
# or
npm run run:android                # Android Emulator
```

Once the app is running:

1. Tap **"Aggiorna widget"** in the app. That fetches the mock and calls `fatturatoWidget.updateTimeline([...])`.
2. Long-press the Home Screen, add the **Fatturato** widget from the "Fatture in Cloud Widget" app.
3. The widget shows the mocked revenue and the top upcoming deadlines.

The mock is deterministic — every refresh returns the same values with an updated `updatedAt` timestamp.

## How data flows

1. The app calls `getFatturatoAttuale()` and `getScadenzeImminenti()` on the mock over HTTP.
2. `buildWidgetProps(...)` turns the DTOs into strings formatted with `Intl` in `it-IT`.
3. `fatturatoWidget.updateTimeline([{ date, props }])` pushes those props into the native widget bundle.
4. `expo-widgets` bridges the props to the SwiftUI / Jetpack Compose surface generated at prebuild time. The widget renders using `@expo/ui/swift-ui` components; no Swift or Kotlin is authored by hand.

## Non-goals

- Real Fatture in Cloud API integration or OAuth.
- Live Activities / Dynamic Island.
- Interactive widgets (buttons, toggles).
- Widget configuration UI (`IntentConfiguration`).
- Production builds, EAS, TestFlight, Play Console.
